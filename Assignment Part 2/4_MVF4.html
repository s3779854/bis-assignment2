Minimal Viable Feature 4: Inventory system

Inventory system (Collection and use of objects)


User Story

Story1: Track the items

  • As a: user
  • I want: to track the inventory
  • So: the inventory shows the items with description

Acceptance criteria:

Criterion 1:

  • Given that the user is in the game.
  • When the user press “B” on the keyboard.
  • Then the game should show the inventory.

Criterion 2:

  • Given that the inventory has shown.
  • When the item is selected (by the keyboard arrow).
  • Then the description of the item shown.

Story2: Collect the items

  • As a: user
  • I want: to collect the items over the game.
  • So: the item is added to my inventory.

Acceptance criteria:

Criterion 1:

  • Given that user is walking over an item has not collected before.
  • When user press “L” on the keyboard.
  • Then user can see a new box with the item’s image in the inventory.

Criterion 2:

  • Given that user is walking over an item has collected at least one before.
  • When the user press “L” on the keyboard.
  • Then the user can see the number on the right bottom of that item box has increased.

Story3: Use the items

  • As a: user
  • I want: to use the item in my inventory.
  • So: I can use the item according to its function.

Acceptance criteria:

  • Given that the user tracks the items in the inventory.
  • When the user selects the item needed to use and press “o” on the keyboard.
  • Then the item can be use according to its function.

    Criterion 1: If the item is a weapon, then convert the current weapon to that and put the current weapon into inventory.

    Criterion 2: If the item is a pill, then decrease the number of “pills” item in the inventory. When the number turns to 0, that item disappears.

    Criterion 3: If the item is a key, then unlock the door needed to open and the item disappears.
