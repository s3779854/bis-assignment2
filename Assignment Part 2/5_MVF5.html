Minimal Viable Feature 5: Sound Design

Visual representation of sound triggers


Player input:

Input Type of sound played
Move key pressed Footsteps
Jump key pressed Woosh/springy/bouncy
Draw weapon key pressed Click/tacky/swish (or depending on weapon)
Attack key pressed
  • Thump/slap for melee
  • Liquid squirt/spray for hand sanitizer
Interact key pressed (door unlock) Key opening door

Player situations:

Circumstance Type of sound played
Main character takes damage Bone/branch crack
Main character dies Oof/ugh
Main character picks up weapon Click/tacky/swish (or depending on weapon)
Main character picks up armor Armor clank

Enemies:

Circumstance Type of sound played
Enemy character takes damage Thump/crack to better indicate that damage is registered
Enemy character dies Oof/ugh (more dramatic if enemy is a boss)
Enemy walking Footsteps or something more alien sounding (distorted/8-bit/digital)
